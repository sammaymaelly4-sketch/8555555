## 2026-03-19 - [Cache API Requests for Image Resolution]
**Learning:** The `resolveImage` utility (duplicated across App.jsx and Login.jsx) fetches image URLs from an external API (OpenFoodFacts) on every component mount without caching. Since multiple components (`ProductCard`, `Produto`) can render the same product simultaneously (e.g., in lists and carousels), this caused redundant, parallel network requests for the same product barcodes.
**Action:** Implement Promise-based in-memory caching for API requests that fetch static or rarely-changing data. By caching the Promise itself rather than just the result, we prevent race conditions where multiple components trigger the same request before the first one completes. Always check if utility functions that make network calls are being used in lists or loops.
